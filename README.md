# JonOps — Autonomous Marketing Agent for Small Businesses

An AI-powered autonomous marketing system for small businesses. A single Claude Code agent runs on a cron schedule inside a Docker container, executing daily marketing tasks: content creation, SEO, email management, social media, newsletter campaigns, and outreach — all without human intervention.

**Works for any business:** bakery, landscaper, lawyer, accountant, astrology site, fitness studio, beauty brand, e-commerce store, or any content-driven small business.

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/jonjonesai/templatized-jonops.git /opt/jonops
cd /opt/jonops

# 2. Run the initialization wizard
npx @anthropic-ai/claude-code /init

# 3. Set up your API keys
cp .env.example .env
nano .env

# 4. Create Airtable tables
python3 scripts/setup/create-airtable-tables.py

# 5. Launch
docker compose up -d
```

The `/init` wizard asks ~15 questions about your business and automatically generates your configuration files.

**See [QUICKSTART.md](QUICKSTART.md) for the 15-minute setup guide.**

## How It Works

A Python scheduler (`scheduler.py`) reads `schedule.json` and launches the Claude Code CLI at designated times. Each run loads a **skill file** — a detailed prompt that tells the agent exactly what to do, which APIs to call, and how to report results. The agent executes, logs to Airtable/Asana, sends a Telegram alert, and exits.

```
schedule.json → scheduler.py → Claude Code CLI → skill file → APIs → done
```

## Deployment

### Option 1: Use the Wizard (Recommended)

1. **Clone the repo:**
   ```bash
   git clone https://github.com/your-org/templatized-jonops.git
   cd templatized-jonops
   ```

2. **Run the initialization wizard:**
   ```bash
   claude /init
   ```
   The wizard asks about your business, brand voice, integrations, and generates your config files.

3. **Set up API keys:** Copy `.env.example` to `.env` and fill in your credentials. See `SETUP.md` for detailed instructions on getting each API key.

4. **Set up Airtable:**
   ```bash
   python3 scripts/setup/create-airtable-tables.py
   ```

5. **Launch:**
   ```bash
   docker compose up -d
   ```

6. **Verify:**
   ```bash
   python scheduler.py --status
   ```

### Option 2: Manual Configuration

1. Copy `templates/CLAUDE.template.md` to `CLAUDE.md` and fill in the placeholders
2. Copy `templates/email-persona.template.md` to `email-persona.md`
3. Copy `templates/MEMORY.template.md` to `MEMORY.md`
4. Continue from step 3 above

## Airtable Setup

JonOps uses Airtable as the source of truth for 19 tables covering content, social, email, and outreach pipelines.

**Quick setup:**

1. Create a new Airtable base at [airtable.com](https://airtable.com)
2. Get your credentials:
   - Personal access token: [airtable.com/create/tokens](https://airtable.com/create/tokens)
   - Base ID: From your base URL (starts with `app...`)
3. Set env vars in `.env`
4. Run: `python3 scripts/setup/create-airtable-tables.py`
5. Copy the output table IDs into your `.env`

See `SETUP.md` for detailed Airtable setup instructions.

## Telegram Chat Interface

Talk to your agent directly from Telegram — like texting a team member. Ask questions, assign tasks, review content, all from your phone.

```
You: "What did we publish today?"
Agent: "Today we published: [lists blog posts, social posts, emails]"

You: "Write a social post about our spring sale"
Agent: [writes and queues a social post]

You: "Check if anyone replied to that Reddit thread about landscaping"
Agent: [checks and reports back]
```

**Features:**
- Same context as scheduled skills (reads CLAUDE.md, uses all tools)
- Session persistence — agent remembers previous conversations
- Daily token budget to control costs
- Security whitelist — only your chat ID accepted

**Setup:** Set `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, and optionally `TELEGRAM_DAILY_TOKEN_BUDGET` in `.env`. The daemon starts automatically with the container.

**Commands:**
- `/reset` — Start a fresh conversation
- `/status` — Show token usage and session info

See `TELEGRAM-INTERFACE-SPEC.md` for technical details.

## Customization

### Disable Skills You Don't Need

Edit `schedule.json` and set `"enabled": false` for any skill:

```json
{
  "skill": "b2b-outreach-lead-finder",
  "schedule": "daily",
  "time": "14:00",
  "enabled": false
}
```

### Adjust Timing

All times are in your configured timezone. Edit times in `schedule.json`:

```json
{
  "skill": "daily-contribution",
  "schedule": "daily",
  "time": "06:00",  // Changed from 07:00
  "enabled": true
}
```

### Add Custom Skills

Create a new skill file in `.claude/skills/` following the existing format:
1. YAML frontmatter with metadata
2. "What This Skill Does" section
3. Step-by-step execution instructions
4. Error handling
5. SKILL_RESULT format

## Environment Variables

All credentials are injected via Docker environment variables. Required vars:

```bash
# WordPress
WP_URL=https://yourdomain.com
WP_USERNAME=your_wp_user
WP_PASSWORD=your_wp_app_password

# Airtable
AIRTABLE_API_KEY=pat_xxx
AIRTABLE_BASE_ID=appXXX
# Plus all AIRTABLE_*_TABLE vars from setup script

# Image Generation
REPLICATE_API=r8_xxx
TINIFY_API=xxx

# Notifications (recommended)
TELEGRAM_BOT_TOKEN=xxx
TELEGRAM_CHAT_ID=xxx
```

See `.env.example` for the complete list based on your enabled integrations.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Skill fails with "missing env var" | Check `.env` has all required variables |
| WordPress publish fails | Verify WP app password and user permissions |
| Airtable "table not found" | Run the setup script and copy table IDs to `.env` |
| Telegram alerts not working | Verify bot token and chat ID |
| Agent not running on schedule | Check `scheduler.py` is running: `ps aux | grep scheduler` |
